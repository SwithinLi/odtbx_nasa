OD Toolbox 4.0 Release Notes

November 30, 2010

 

The following sections describe the New Features, the Changes to Existing Features, the Known Bugs, and Notes in OD Toolbox Release 4.0 since the prior 3.5 release.

 

 

NEW FEATURES:

 

Estimator Functions:

 

·         A square root information smoother (SRIS) was added to the existing square root information filter, estsrif.  For more information see the estsrif help or go to MATLAB Help: Orbit Determination Toolbox: Supporting Documentation: “Square-Root Information Filter and Smoother, Huang, 2010”.  See also the UseSmoother option in odtbxOptions.

 

·         A “restart record” capability was added to estseq.  This allows a user to save the entire end state of an estseq run, modify it, and then use it to reinitialize a future estseq run. One specific application is to allow the user to simulate an impulsive maneuver by manipulating the restart record in between two separate estseq runs.  As a useful example, the new impulsiveBurn applies an impulsive delta-v to the state and covariance (linear and Monte Carlo) stored in a restart record structure.

 

·         estseq and estsrif now have the ability to perform vector measurement editing rather than only performing measurement editing on a scalar basis.  A new option, EditVector, was added to odtbxOptions.  Several ODTBX support functions (chi2cdf_odtbx, chi2inv_odtbx, and chi2pdf_odtbx) were introduced to compute the chi-squared distribution statistics required by vector measurement editing without requiring the Statistics Toolbox.

 

·         estspf now has the ability to edit measurements.

 

Measurement Functions:

 

·         Added xnavmeas, a measurement model that models the range difference and range difference rate between a spacecraft and a distant inertially-fixed Resident Space Objects (RSOs).  This is the public version of the XNAV measurement model developed by Microcosm.  For more information see the MATLAB Help: Orbit Determination Toolbox: Supporting Documentation: “XNAVMEAS Range and Range Rate Measurement Function for ODTBX, 2010”.

 

·         Added getPulsarData, which retrieves a library of pulsar data for use with xnavmeas.

 

·         Added a schedule tracking capability to xnavmeas.  A tracking schedule restricts the xnavmeas measurement model to only provide measurements during specific time intervals.

 

·         createGroundStationList now accepts full pathnames to a file as an argument even if the file is not on the MATLAB path.

 

·         testMeasPartials was added to help validate the measurement partials matrix of a measurement model.  It compares an analytic calculation to a numerical approximation.  The testMeasPartials_example demonstrates its use with gsmeas, tdrssmeas, ddormeas, and lnrmeas.

 

Other Functions:

 

·         Added HermiteInterpolator, a divided difference Hermite interpolator of arbitrary order which does not require equal time steps.  Also implemented a version of this in JAT for use with the jatWorldPropogatorRK* JAT adaptors.

 

·         Provided XNAV_only_LEO.m, an example of a LEO orbit determination simulation using xnavmeas.

 

·         Provided gpsmeas_example.m, an example of using gpsmeas with an orbiting GPS receiver.

 

 

CHANGES TO EXISTING FEATURES:

 

Estimator Functions:

 

·         estseq, estsrif, and lincov_kf now pass options to the integrator, integ.

 

·         Estseq scalability performance when running multiple Monte Carlo runs with the Parallel Computing Toolbox has been improved 375%.

 

·         Added protections to estspf to handle TSPAN as a column vector.

 

·         estbat can now accept a time specification of the form TSPAN = [T0 TFINAL], which allows the integrator to set the time step of output data.

 

·         Bugfixes in estsrif and estseq ensure the estimation error at t=0 is populated correctly.

 

·         A bugfix in estsrif ensures the process noise covariance is positive definite during processing.

 

·         Bugfixes in estbat, estseq, and estspf corrected problems handling NaN measurements.

 

Measurement Functions:

 

·         A bugfix for tdrssmeas now allows use of a measurement schedule with TDRSS measurements.

  

Force Modeling Functions:

 

·         Bugfixes were made to the JAT force models.  The most significant performance changes in ODTBX functions are in jatForces, jatForces_km, and jatChargedParticleModel.

 

Other Functions:

 

·         Improved the performance of getOdtbxOptions, which is used in many functions.

 

·         Improved the plotting capabilities of plot_ominusc.  It can now plot multiple Monte Carlo cases correctly and plot edited measurements.

 

·         Updated the version of MICE, the MATLAB interface to NAIF SPICE, to version N0064, released June 11, 2010.  The following platform installations are supplied with this ODTBX release:

·         Linux x86 64-bit

·         Mac Intel 64-bit

·         Windows 32-bit

·         Windows 64-bit

If required, other MICE versions or platforms could be used with MATLAB and ODTBX.  See Notes, below.

 

·         Several discrepancies in the odtbxOptions have been resolved.

 

 

KNOWN BUGS:

·         Problems with a dual-monitor setup in MATLAB with Windows:

An existing MATLAB bug may affect the way estval and some other ODTBX plots routines display the plots when using dual monitors under Windows.  The MATLAB bug is #301358 (Mathworks account required) and occurs in all recent MATLAB versions, R14 (7.0) through 2010b (7.11) and may affect different graphics drivers differently.

  

 

NOTES:

·         Other MICE versions: If required, a different version of MICE can be used with MATLAB and ODTBX than the ones supplied with this ODTBX installer.  Frequently this is required when a particular platform is not supported by the ODTBX installer.  MICE and SPICE can be downloaded from http://naif.jpl.nasa.gov/naif/toolkit.html.  Install MICE into a directory under the ODTBX installation directly and name it “mice”.  The startup.m supplied with ODTBX will look for MICE paths in that directory.  ODTBX paths also expect a “mice/kernels” subdirectory as a convenient place for SPICE kernels to live.  Create that subdirectory as well, even if you don’t intend to use it, so that all of the ODTBX MICE paths will be found.  You will have to download your own SPICE kernels but the pck00009 (or later), the naif0009 (or later), and the de421 kernels are recommended.
