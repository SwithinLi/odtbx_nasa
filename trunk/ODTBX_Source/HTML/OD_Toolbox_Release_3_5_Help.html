OD Toolbox 3.5 Release Notes

May 30, 2010

 

The following sections describe the New Features, the Changes to Existing Features, and the Known Bugs in OD Toolbox Release 3.5.

 

NEW FEATURES:

 

Estimator Functions:

 

·      A Square Root Information Filter – estsrif

 

The corresponding math specifications can be found in Orbit Determination Toolbox >  Supporting Documents.  An example of its use can be found in the Examples folder.

 

·      A separate linear covariance analysis function - lincov_kf

 

This functionality used to be internal to estseq.  Now both estseq and estsrif call this function to perform the covariance analysis. 

 

·      estseq now supports numerical computation of measurement partials for use in the covariance analysis.

 

·      estseq has significantly improved its accuracy by the implementation of the variable step integration of the states and their covariance.

 

·      A new odtbxOptions parameter, ‘refint,’ has been created that specifies how the time points are assigned between measurement updates.

 

Measurement Functions:

 

·      Ground stations in gsmeas can now be specified through their ECEF coordinates thereby bypassing calls to the ground station database which can significantly improve the speed of simulations involving gsmeas.

 

·      The rate of change of the delta-difference range measurement has been added to the measurement function ddormeas.  This could be used in place of Doppler in certain circumstances.

 

·      Additional antenna pointing options in gpsmeas have been enabled:

 

o    (2) parallel, or (-2) antiparallel to the Earth-Sun vector

o   (4) fixed with respect to apogee zenith, or (-4) nadir vector apogee selected as the point of highest altitude (ephemeris must include apogee)

o   (5) body fore and (-5) aft directions relative to geocentric LVLH

o   (6) body port and (-6) starboard directions relative to geocentric LVLH

 

·      The function comp_bs_3d has been made more accurate by the replacement of the sidereal transformations with ECI-ECEF transformations.

 

·      A new option ‘PrecnNutnExpire’ has been added to odtbxOptions which the function gpsmeas now looks for that sets the time period between recomputations of the precession and nutation when converting between ECI and ECEF.  The default is 1 day.

 

·      The function kepanom now recognizes both ‘nu’ and ‘tran’ as a true anomaly field in the orbit element structure.

 

Other Functions:

 

·      The function jatDCM has been made faster by ~80% when computing transformations between ECI and ECEF.  A new parameter has been added to indicate the time period in which the precession and nutation can be kept constant before being recomputed.

 

·      The following JAT time conversion functions have been made more efficient:

 

o   jD2MJD.m

o   jD2MatlabTime.m

o   matlabTime2JD.m

o   matlabTime2MJD.m

o   mJD2JD.m

o   mJD2MatlabTime.m

 

CHANGES TO EXISTING FEATURES:

 

Estimator Functions:

 

·      The default values of ‘EditFlag’ and ‘EditRatio’ in odtbxOptions have changed from not performing measurement edits to performing edits with a 3-sigma edit ratio.

 

·      Better handling of NaN measurements in kalmup has been implemented.

 

·      An error will now be issued if Po is not specified in estspf or estseq. 

 

·      The time vector is now correctly populated when the option refint=1 in estseq.

 

·      The input time vector for estseq can now be specified as a row or a column array.

 

·      The covariance associated with a measurement innovation is now correctly computed in ominusc and kalmup when excluding NaN measurements.

 

·      The function calculatetrueanomalystep has been removed.

 

·      Zero innovations in estspf have been replaced with NaNs.

 

·      Corrected the 'randnarray' typo in covsmpl

 

Measurement Functions:

 

·      The read_yuma function now takes an optional argument of the simulation date and rolls over the GPS almanac week to get as close to that date as possible.

 

·      tdrssmeas now sets the correct ground station when t(1) ~=0

 

Force Modeling Functions:

 

·      Corrected a transformation and an interpolation error in JAT atmospheric drag computation

 

·      Corrected a coding error in the calculation of the Sun ephemeris

 

·      Changed the default ‘RelTol’ from 1e-3 to 1e-9 in jatWorldPropagatorRKF78

 

Other Functions:

 

·      Corrected an incorrect value of Earth rotation rate in JatConstant

 

·      Added comments in odtbxOptions to highlight the valid altitudes for the atmospheric models. The 'useAtmosphericDrag' option in the odtbxOptions structure is functional within these valid ranges of the atmospheric models.

 

KNOWN BUGS:

None
