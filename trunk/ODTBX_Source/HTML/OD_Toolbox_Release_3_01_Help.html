OD Toolbox Release 3.01 Release Notes

January 8, 2010

OD Toolbox Release 3.01 is a minor release, primarily to release the software under the NASA Open Source Agreement (NOSA). In addition, the following changes have been made to Release 3.0:

  • Corrected the 'randnarray' typo in covsmpl

  • Changed the default RelTol from 1e-3 to 1e-9 in jatWorldPropagatorRKF78

  • Added comments in odtbxOptions to highlight the valid altitudes for the atmospheric models. The 'useAtmosphericDrag' option in the odtbxOptions structure is functional within these valid ranges of the atmospheric models.

  • Included some ODTBX reference papers in the Matlab Help Browser under Orbit Determination Toolbox > Supporting Documentation



OD Toolbox Release 3.0 Release Notes

November 23, 2009


What's New in Release 3.0

The following sections describe the New Features, the Changes to Existing Features, and the Known Bugs in OD Toolbox Release 3.0.


New Features:

  • Matlab interface to SPICE, the CSPICE library using the Matlab interface "MICE" developed by JPL.

MICE documentation can be accessed via the Matlab help system. Proper version of Mice is installed based on the user's platform or the user's option. SPICE is designed to access kernel data most of which are very specific to a given application. This provides SPICE with the ultimate flexibility to continuously add new kernels. The kernels can be found from the JPL source ftp://naif.jpl.nasa.gov/pub/naif/. Three of these kernels, making up a standard kernel set, have been packaged with the ODTBX 3.0 installer and are located in the mice/kernels folder. These may be loaded with the cspice_furnsh function (making sure to provide the full path name to the kernels in quotes). These three kernels are:

    1. de421.bsp - SPK Planetary ephemeris kernel

    2. naif0008.tls - LSK Leapseconds kernel

    3. pck00008.tpc - PCK Planetary constants kernel

  • Attitude dependent solar radiation pressure via a flat plate model - srpAccel

A line-of-site calculation is included to turn off solar radiation pressure accelerations in case of shadow from the earth and/or the moon. A check on the distance from the Sun was added to the original model to make the penumbra calculations more realistic.

  • Utilities for simulating the Deep Space Network (DSN)

The following new Matlab functions and new adaptors to existing JAT functions were created:

    1. jatStaAzEl computes the azimuth and elevation of a satellite relative to a ground station. (This replaces the previously less accurate elevation.)

    2. EarthOrbitPlot plots 3-D Earth orbit trajectories.

    3. stationData provides access to monthly average temperature, pressure, relative humidity and temperature lapse for the DSN stations for use in tropospheric delay models.

    4. ttdelay models time tag delays.

    5. relativityLightDelay models the effects of relativity on DSN measurements.

    6. staEarthTide provides a model of the solid Earth tide effects on DSN station locations.

    7. jatIonoDelayModel computes the ionospheric delay.

    8. jatTropoModel computes tropospheric delay.

    9. jatChargedParticleModel computes light time delay error due to charged particle model.

  • Schmidt-Kalman Filter estimator

This functionality was implemented in estseq and can be invoked with the odtbxoptions option SchmidtKalman. The mathematical background on Schmidt-Kalman filter can be found in Chapter 9 of Brown, R. G. and P. Y. Hwang, Introduction to Random Signals and Applied Kalman Filtering, With Matlab Exercises and Solutions, Third Edition, John Wiley & Sons, 1997.

  • Additive divided-difference sigma point filter - estspf

The algorithm follows the paper: Carpenter, J. R., S. Hur-Diaz, and F. L. Markley, "Generalized Covariance Analysis of Additive Divided-Difference Sigma-Point Filters," Paper AAS 09-305 at the AAS/AIAA Astrodynamics Specialist Conference, Pittsburgh, PA, 9-13 August 2009.

  • TDRSS based measurements - tdrssmeas

Modified gsmeas to include a return link as well as Doppler measurements. TDRSS scheduling capabilities are also implemented.

  • Lunar relay based measurements - lnrmeas

  • Utility to input STK ephemeris - stkephem

  • Improved GPS measurement functionality - gpsmeas

A thorough cleanup and verification of this function was performed to improve its speed and accuracy. The following list summarizes the major changes:

    1. Created dataCache to cache Yuma almanac data file reads over multiple gpsmeas calls. Approximately a 30% gain in speed was realized.

    2. Removed all but one eval statement

    3. Cleaned up all mlint warnings

    4. Corrected bugs related to time specification in a coordinate transformation

    5. alm2xyz issues a warning if the ephemeris is too old or too far ahead in the future by 7 days.

    6. Added the Ecliptic North/South Antenna Pointing option

    7. Sample GPS antenna files can be found in ODTBX_Data/antennae folder.

    8. Only GPS block II/IIA data is publicly available.

  • One-way range measurement function - rrdot3D_1way

  • Unit line of sight vector measurement - LOSunit

  • Utility to plot measurement residuals or innovations - plot_ominusc

  • Delta Differenced One-way Range (DDOR) measurement - ddormeas

DDOR is a measurement technique where two widely separated antennas will track a transmitting satellite simultaneously in order to measure the delay time between signals arriving at the two stations. This step alone produces a Differential One-way Range (DOR) measurement. Theoretically, the delay depends only on the positions of the two antennas and the spacecraft, but in reality, the delay is also affected by the troposphere, ionosphere, and clock instabilities at the ground stations. Delta-DOR corrects these errors by tracking a quasar in a direction close to the spacecraft for calibration. The chosen quasar's direction is already known extremely accurately by astronomical measurements, typically to better than a nanoradian.

  • Parallel processing of Monte Carlo simulations where JAT force and/or JAT measurement models are used

All three estimators (estseq, estspf, and estbat) were modified so that they can run in a parallel environment. All JAT classes associated with ODToolboxJATModel were made serializable so that they can work in a parallel environment


Changes to Existing Features:

Force Models

  • Added a separate kilometer version of the jatForces, called jatForces_km.

  • jatForces (and jatForces_km) now allows vectorized inputs, i.e., states specified over multiple time steps.

  • ephemDE405 now performs a data bounds-check on the time argument and throws an IllegalArgumentException if the requested time range is outside of the valid range. Furthermore, it now allows the user to provide time system as an optional input by using convertTime to convert it to TT before calling the JAT DE405 function.

  • The A matrix calculation for jatForces is now suppressed and outputs an empty matrix. This forces the estimator to numerically estimate the matrix when the requested model is other than Earth 2-body and J2. For example, this allows the use of the lunar gravity.

Numerical Integrators

  • For Release 3.0, some of the integrators have been renamed or deprecated. The available integrators are now as follows:

    • jatWorldPropagatorRK4 (renamed from jatRK4)

    • jatWorldPropagatorRK8 (renamed from jatRK8)

    • jatWorldPropagatorRKF78 (renamed from jatRKF78)

    • jatRK8 (renamed from jatRungeKutta8)

The first three were renamed to emphasize that these integrators are used with the jatWorld and hence only work with the jatForces or jatForces_km equations of motion. The new jatRK8 requires setJatRK8Options and getJatRK8Options for its specific options structure. These options are not the same as odtbxOptions.

  • jatWorldPropagatorRKF78 now allows for input time array length to be less than 8, the number required for the internal Lagrangian interpolator. If the number is less than 8, linear interpolator is used.

  • RK4 is not supported anymore and is replaced by jatRK8.

Constants

  • In JATConstants, the incorrect value of the Moon mean radius was updated to 1738 kilometers.

Equations of Motion

  • jatHCW now allows for a user-specified process noise and dynamic function argument parameters to be either an array or a cell.

Coordinate Transformations

  • ecef2lla now has a more accurate iterative method using Newton's method with the exact formulation of lla2ecef.

Measurement Models

  • LOSRange, LOSRangeRate, and LOSDoppler now replace range, rangeRate, and Doppler, respectively.

Ground Station Functionality

  • createGroundStationList now calls NASA_GroundStationList instead of GroundStationList.

Universal Options Structure

  • The following new options have been added to odtbxOptions:

    • SchmidtKalman – runs estseq as a Schmidt Kalman filter

    • EarthAtmMaskRadius – set mask in tdrssmeas

  • Implemented the following formerly identified but non-functioning options:

    • EditRatio

    • EditFlag

    • UpdateVectorized

    • MonteCarloSeed

  • Removed the following unnecessary or unvalidated options:

    • MeasurementSeed

    • CovarianceSeed

    • MeasOpts

    • ForceOpts

    • useGPS

    • useClockError

    • useMeasurementNoise

    • useAntenna

    • outputInterval

    • displayWarnings

    • dopplerType

  • All former uses of ESTSET/ESTGET were converted to the odtbxOptions structure.

Estimators

  • Handling of NaN measurements in the Monte Carlo as well as the covariance analysis sections of estbat, estseq, estbat were corrected or improved.

  • estbat changes:

    • No longer crashes when consider covariance analysis is not being done and datarg is a structure

    • Now accepts a mix of finite and infinite values for the initial covariance matrix (Corresponding updates to covsmpl were made as well.)

    • Now allows for numerical estimation of the measurement Jacobian

    • A bug in checking NaN measurements in observ was fixed

    • Now allows empty, infinite, or unspecified initial covariance matrix

    • Fixed incorrect a priori conditions for Monte Carlo simulations

    • Has an option to ignore process noise

  • estseq changes:

    • Fixed a bug in the specification of a priori conditions for Monte Carlo simulations

Other Changes

  • kepel can now handle either a 1x3 or a 3x1 input vector.

  • When using functions that require time and pole conversion data stored in FitIERS EOP.dat, the user will now be informed when the data-need outstrips the supplied information files with the JAT adaptor function getIERSTimes.

  • covsmpl now correctly executes with the 'principalAxis' argument.

  • convertTime now handles arrays of time values correctly for all possible to/from pairs.

  • covmake now checks for non-positive covariances.

  • kepprop2b now accounts for non-zero mean anomaly.

  • estval now shows a legend of the various plot elements as well as an option to plot the error sigmas relative to the mean error or zero.

  • Obliquity of the ecliptic calculation for comp_bs_3d replaces the hard-coded value.

  • Leap seconds are now loaded from a data file, leapsec.dat. Updating UTC leap seconds now only involves editing this text file rather than modifying code. Also the missing century and 400 year checks were added to the leap-year calculation algorithm.

  • jatWorld polar motion error has been fixed.

  • Incorrect or obsolete code dependencies or references were removed.

  • General cleanup of help comments for all functions was performed.

  • The ODTBX_Examples folder was scrubbed to remove nonfunctioning, nonvalidated, or outdated examples, and new examples were added.

Regression Testing

  • Regression tests have been provided for all new Release 3.0 functions.


Known Bugs:

  • The 'useAtmosphericDrag' option in the odtbxOptions structure is currently nonfunctional. A new implementation of the drag model is being considered and will be included in Release 4.0.
